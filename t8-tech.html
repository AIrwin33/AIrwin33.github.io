SQL Query Optimization

Andrew Irwin

Having dabbled in SQL earlier this week, I wanted to learn more about how I could use SQl more efficiently. Similar to code refactoring that we learned about in Ruby and JavaScript, SQl query optimization can cut down on the amount of time it takes to run a query, as well as look cleaner and more concise.

So far, we have just learned about SELECT queries, so I'll just focus on those for this blog post.

The first SQL SELECT statement I learned was used to select all fo the columns in a given table. for shorthand, you can use '*' to represent all of the these columns. Instead, type out all of the columns before calling FROM and the relevant table.

Another example is when using subqueries. Look at the query below. Is there any way to pare this down?

				
				SELECT name
				FROM employee
				WHERE salary = (SELECT MAX(salary) FROM
				employee_details)
				AND age = (SELECT MAX(age) FROM
				employee_details)
				AND emp_dept = 'Electronics';

What if we combined the salary and age subqueries? Afterall, they both asking for something from the same information (employee_details)?

				
				SELECT name
				FROM employee
				WHERE (salary, age) = (SELECT MAX(salary), MAX (age) FROM
				employee_details)
				AND emp_dept = 'Electronics';

Here are a few more tips:

Use BETWEEN instead of max/min

use '%' when looking for words that contain certain characters. Ex. LIKE 'Char%' finds any words that begin with 'Char'.

Try not to use !=, do use <,>, etc.
